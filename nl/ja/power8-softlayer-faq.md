# {{site.data.keyword.Bluemix_notm}} での POWER8 - FAQ

|1.|オファリングの詳細||
|---|---|---|
|1.1|何がいつ行われますか?|<ul><li>SoftLayer のオファリングが拡張され、POWER8 ベースのベア・メタル・システムが含まれるようになりました。これらのシステムは、IBM の新しい POWER8 プロセッサーおよび OpenPOWER ベースのプラットフォームを使用して構築されています。このプラットフォームは、Linux でデータ、コグニティブ、Web のワークロードをクラウド・ベースでデプロイするために特別に調整されています。</li><li>VMaaS (サービスとしての仮想マシン) は、ベータ・テスト中であり、近い将来に提供される予定です。</li></ul>|
|1.2|OpenPOWER とは何で、注目すべきなのはどうしてですか?|<ul><li>OpenPOWER ベースのサーバーは、POWER8 アーキテクチャー上に構築された新しいタイプの強力でカスタマイズ可能なオープン・システムです。POWER8 は、データ集中型のワークロード用に一から設計された唯一のプロセッサーです。</li><li>OpenPOWER Foundation (OPF) は 2013 年 8 月に、大きなアイデアを持ったわずか 5 社から成るコンソーシアムとして小さく開始しました。この大きなアイデアとは、POWER ハードウェアおよびソフトウェアをオープン開発で使用可能にすることで、テクノロジーおよびイノベーションの新しいアプローチを実現すること、および業界全体の提携関係を構築して、そのプラットフォームでのイノベーターのエコシステムを拡大することです。OPF は、その誕生からの短期間で、参加企業が 190 社を超えるまでに成長しました。この急速な成長は、「データ・センターの再考 (Re-imagine the datacenter)」を目指すこの取り組みに対する業界全体の興奮を示しています。</li><li>OpenPOWER プラットフォームは、POWER8 の能力、および OpenPOWER でのオープン・コラボレーションからのイノベーションを SoftLayer パブリック・クラウドにもたらします。</li></ul>|
|1.3|SoftLayer での POWER8 プラットフォームの利点は何ですか?|<ul><li>POWER8 は、従来型のクラウド・プラットフォームに代わる独自の選択肢であり、データ・ワークロードに最適化されたアーキテクチャーを備えています。パフォーマンスのテストでは、同等の x86 システムと比較して、データ・ワークロードのパフォーマンスが平均して 2 倍優れているいることが示されています。ベンチマーキング・テストでは、SoftLayer での POWER8 によって以下が実現可能であることが示されています。<ul><li>DB2 with BLU で、1 時間当たり 1.76 倍多くの照会、1 照会当たり 20% のコスト削減。</li><li>クラウドでの DB2 with BLU Acceleration on POWER8 により、コグニティブの洞察が 43% 高速化、同じ時間で 65% 多くの洞察を提供。</li><li>データ分類および予測分析で使用される Spark 機械学習で 64% 優れたパフォーマンス。</li><li>ビッグデータ・クラスターで広く使用されている Spark SQL 処理で 66% 高速化。</li><li>Spark グラフ・ワークロードに基づいて、ソーシャル接続と製品推奨の実行を 83% 高速化。</li></ul></li></ul>|
|1.4|オファリングの詳細はどのようなものですか?|<ul><li>POWER8 システムの最初のフェーズは、月単位のレンタルで使用可能であり、4 つの固定のベア・メタル・デプロイメント構成 (小規模、中規模、大規模、大規模 (SSD を使用)) があり、Ubuntu OS がインストールされています。</li><li>オファリングには、標準 SoftLayer API サポート、およびストレージ、ネットワーキング、ファイアウォールなどの付属サービス・オファリングのほとんどが含まれています。</li><li>POWER8 システムは当初、DAL09 データ・センターに配置されています。2016 年の後半に他のジオグラフィーに展開される予定です。</li></ul>|

|1.5|命名は何を意味しますか?|新しい POWER8 命名フォーマットは、C812L です。ここで、各英数字は以下を意味します。<ul><li>C = Cloud (クラウド)</li><li>8 = POWER8</li><li>1 = コア数</li><li>2 = 2U</li><li>L = Linux</li></ul>|
|1.6|どのようなオペレーティング・システムがサポートされますか。AIX または IBM i を実行できますか?|<ul><li>SoftLayer では、当初、Ubuntu Linux 14.04 LTS オペレーティング・システムを提供します。今後、他の Linux ディストリビューションを追加する予定です。</li><li>これらのシステムは、AIX や IBM i を実行できません。サービスとしての AIX は、IBM Cloud Managed Services から使用可能であり、ホストされた IBM i は、IBM のグローバル・ビジネス・パートナーの多くから使用可能です。</li></ul>|
||AIX または IBM i オペレーティング・システムを搭載したシステムをレンタルできますか?|<ul<li>IBM では、引き続き、IBM Cloud Managed Services および他のパートナーを介してクラウドでの AIX を提供します。IBM i ベースのクラウド・サービスは、MSP パートナーの多くからも使用可能です。</li></ul>|
|**2.**|**購入**||
|2.1|POWER8 を使用してメリットを享受できるのはどのようなユーザーですか?|<ul><li>SoftLayer での POWER8 初期オファリング・スイートは、ハイブリッド・クラウド・ソリューション用の既存のシステムの価値を拡大したい Power の既存のお客様に良く適しています。</li><li>これらのシステムは、データ・ワークロード用に最適化されており、クラウドでの開発/テストのため、Power のパフォーマンスの利点を実証するための迅速な POC のためや、Power に<a href="https://www.ibm.com/developerworks/library/l-port-linux-on-x86-application/">アプリケーションを簡単に移植する</a>ためや、<a href="https://www-304.ibm.com/partnerworld/wps/servlet/ContentHandler/stg_com_sys_power-development-platform">Power Developer Cloud</a> および <a href="https://www-304.ibm.com/webapp/set2/sas/f/lopdiags/sdklop.html">SDK for Linux on Power</a> 用に簡単に利用できます。</li><li>SoftLayer クラウドでの POWER8 は、プライベート・クラウドとパブリック・クラウドの両方のリソースを利用する統合アプリケーションをサポートするためにも使用できます。</li><li>また、POWER8 ベア・メタル・システムは、ゲノミクス、金融、コグニティブなどのサービスの新規アプリケーション用の優れたホスティング・プラットフォームにすることもできます。</li></ul>|
|2.2|最も一般的なユース・ケースはどのようなものですか?|<ul><li>DB2 with BLU、Cognos、WebSphere など、Ubuntu OS で実行されるデータおよび分析集中型のワークロード。</li><li>Apache Spark、MariaDB などのオープン・ソース・アプリケーション、および  LAMP を使用する e-コマースやコンテンツ管理システムなどのアプリケーション。</li><li>Linux on Power ワークロード用の開発/テスト、移植、調整。</li><li>競争の激しい UNIX ワークロードおよび他のアプリケーション・スタック用の PoC (概念検証)。</li></ul>|
|2.3|どのようなメリットが顧客にもたらされますか?|<ul><li>POWER8 は、データ用に設計された唯一のプロセッサーであり、IBM Power Systems サーバー・ラインの旗艦製品です。{{site.data.keywords_baremetal_short}} は、要求の高いデータ集中型のコグニティブ・ワークロードをクラウドで実行したいと考えている企業のお客様に最適です。</li><li>データ・ワークロードおよび分析ワークロードの場合、POWER8 の増強されたスレッド、メモリー帯域幅、速度により、同等の x86 ベースのシステムよりも高いスループットおよびパフォーマンスを実現できます。</li></ul>|
|2.4|POWER8 {{site.data.keywords_baremetal_short}} はどのように注文しますか?|<ul><li>サービスとしての POWER8 ベア・メタルは、他のすべての SoftLayer サービスと同様にカタログから入手および注文可能になります。</li><li>POWER8 ベア・メタルは、IBM および BP セラーが任意の IBM Cloud 契約に組み入れることができる標準 IBM Cloud オファリングとしても入手可能です。</li></ul>|
|2.5|POWER8 システムはどのような価格設定ですか?|<ul><li>POWER8 ベア・メタル・システムは、提供パフォーマンスに基づいて、x86 システムと同等の価格で提供されます。</li><li><strong>小規模:</strong> 8 コア、64GB RAM、1x1TB SATA、10G Gbps、予備電源。</li><li><strong>中規模:</strong> 10 コア、256GB RAM、2x4TB SATA、10 Gbps、予備電源。</li><li><strong>大規模:</strong> 10 コア、512GB RAM、2x6TB SATA、10 Gbps、予備電源。</li><li><strong>大規模/SSD:</strong> 10 コア、256GB RAM、2x960 SSD、10 Gbps、予備電源。</li></ul>|
|**3.**|**SoftLayer での Power の実装**|||
|3.1|多層アプリケーション環境のために、これらの新しい POWER8 システムを SoftLayer 内の隣接の x86 プラットフォームに接続できますか?|<ul><li>はい。これらのシステムは、Power と x86 の両方のテクノロジーを必要とする多層アプリケーション環境のために、他の SoftLayer オファリングおよびアプリケーションに隣接するように構成できます。</li></ul>|
|3.2|オペレーティング・システムなしで POWER8 システムをプロビジョンできますか?|<ul><li>現時点では、「OS なし」を実行するオプションは使用可能ではありません。</li></ul>|
|3.3|RHEL や SUSE など、独自の Linux オペレーティング・システムを持ち込むことはできますか?|<ul><li>はい。ただし、そのような構成はサポートされず、また検証もされていません。ソフトウェア・ベンダーとの間でのライセンス管理、パッチ、使用料、保守に対しては、お客様が責任を持つことになります。SoftLayer はハードウェアをサポートし続けます。ハードウェアの問題が発生した場合には、ハードウェア障害に関連した syslog やエラー・メッセージを取得するためのアクセス権限をお客様に求めることがあります。</li></ul>|
|3.4|POWER8 アーキテクチャーでの仮想マシン (VMaaS) は、いつ使用可能になる予定ですか?|<ul><li>POWER8 での VMaaS は、2016 年後半に一般使用可能になる予定です。</li></ul>|
|3.5|どのような自動化およびプロビジョニング・ツールが POWER8 で使用可能ですか?|<ul><li>Canonical の JuJu Charms により、Ubuntu OS へのデプロイメントを自動化できます。Ubuntu への多くの IBM およびオープン・ソースのパッケージのデプロイメントを自動化するための多くの事前にビルドされたパッケージが使用可能です。Canonical の <a href="https://jujucharms.com/store">JuJu Charm Store</a> から入手可能です。</li></ul>|
|3.6|高速プロビジョニング用に設計されたこれらのシステムは、アップグレード可能ですか? また、構成可能なベア・メタルを今後提供する予定ですか?|<ul><li>SoftLayer では、構成オプション、アドオン・サービス、オペレーティング・システムの選択肢、時間単位の価格設定を今後追加する予定です。</li></ul>|
|**4.**|**将来の方向性および追加情報**||
|4.1|これらの新規 POWER8 システムでどのような追加のデータ・センター・インフラストラクチャーおよびサービスが使用可能になりますか?|<ul><li>SoftLayer では、x86 で現在提供しているものと同じデータ・センター・オプションを POWER8 にも提供します。ただし、いくつか例外はあります (eVault およびソフトウェア・ベースのファイアウォール)。</li><li>SoftLayer では引き続き、追加の構成およびストレージ・オプション、オペレーティング・システム、他のデータ・センターへの拡張、GPU、オファリングの選択および能力を拡張する他のオファリングなど、POWER8 の構成およびオファリングを徐々に拡張していきます。</li></ul>|
